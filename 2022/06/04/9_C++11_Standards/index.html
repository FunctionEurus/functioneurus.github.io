#9 C++11标准

2022-06-04
  • C++

Auto关键字

在强类型语言传统C和C++中，参数的类型都必须明确定义，而javaScript，Python等高级语言属于弱类型语言。

优缺点比较：安全性，运行速度、友好、便利性，开发速度。

传统上：auto、register关键字，变量的存储类型修饰不能被单独使用，否则编译器会给出警告。

在C++11标准中，引入关键字auto，decltype，以实现类型推导特性。

auto新用法：使用auto定义变量，不再使用其它类型修饰符修饰，该变量的类型由编译器根据初始化数据或变量自动推导确定。

//根据赋值推断
auto i = 0;   // i被推导为整数类型
i++;             // 整数加法
cout << i << endl;    // 输出1
cout << typeid(i).name() << endl;  // 输出 int

auto  j = "abcde";  //  j被推导为const char *
j++;     //执行指针的位移运算
cout << j << endl;  //输出bcde
cout << typeid(j).name() << endl; // 输出const char *

int arr[3] { 1, 2, 3 }; // 数组名即指针，且地址已经分配
int *pp = arr;   //  pp 即起始地址，pp[1],pp[2] 等均已分配
auto pi = arr;   // pi 是int*
cout << typeid(pi).name() << endl; // 输出int*

//注意推导是在编译期完成的，并不影响性能。
//指定可能发生的异常。

//根据其他变量推断
auto a = 1;
auto pa = new auto(3); // pa 为 int *
auto i = a;   // i 为int
auto *pi = &i; // pi 为 int *
auto &ri = i;  // ri 为 int &
auto j{2};   // j为int
auto k{3}, & r = k;  // k 为int
auto m{4}, n = 5;   // m,n 均为int
auto s{5}, t{0.6}; // 前后类型不一致
auto u;                   // 无法推导出类型
auto *v;                 // 无法推导出类型
auto arr[] {7, 8, 9};     // 不能用于推导数组类型

//auto 不能用于函数参数

int add(auto x, auto y) {
}

更有用的用法

常规用法：

vector<int> ivec(10, 1); // ivec内为10个1

for (vector<int>::iterator it = ivec.begin(); it != ivec.end(); it++) {
    cout << *it << endl;
}

更好的用法：

vector<int> ivec(10, 1);

for (auto it = ivec.begin(); it != ivec.end(); it++) { // 遍历
    cout << *it << endl;
}

Decltype

auto关键字只能对变量进行类型推导，decltype则可对表达式进行推导，用法和sizeof很相似。

auto x = 1;
auto y = 2;
decltype(x + y) z; // z 是int 类型

尾返回类型

考虑模板函数：

template<typename R, typename T, typename U>

R add(T x, U y) {
    return x + y;
}

在使用该模板函数的时候，必须明确指出返回类型。但事实上程序员并不知道add()这个函数会做什么样的操作，获得一个什么样的返回类型。

但是也不能这样定义：

decltype(x + y) add(T x, U y);

因为编译器读到x+y时，x和y还未定义。

C++11的解决方案是使用尾返回类型：

template<typename T, typename U>

auto add(T x, U y) -> decltype(x + y) {
    return x + y;
}

这样就可以使用该模板函数了：

int res = add<int, int>(3, 5);

在C++14中，则支持更直观的语法：

auto add(T x, U y) {
    return x + y;
}

区间迭代

visual Basic、C#、Python等语言中foreach语法的使用。

  1. 遍历数组
int arr[] = {1, 2, 3, 4};   // 数组

for (auto i : arr) {
    std::cout << i << std::endl;
}
  1. 遍历字符串
std::string str = "hello, world";  // 字符串

for (auto ch : str) {
    cout << ch << endl ;
}
  1. 遍历容器
vector<std::string> str_vec = { "i", "like",  "google"};   // 容器

for (auto it : str_vec) {
    cout << it << endl;
}

迭代变量的赋值方式： 引用或拷贝，语法大为简化。

int arr[] = { 1, 2, 3, 4 };   // 遍历数组
int brr[] = { 1, 2, 3, 4 };   // 遍历数组
for (auto &i : arr)     // 可读可写
    i *= 2;
for (auto i : brr)    // 可读不可写
    i *= 2;
for (auto j : arr)
    cout << j << endl;
for (auto j : brr)
    cout << j << endl;
//输出结果：24681234

Default

在传统C++中，如果程序员没有提供，编译器会默认为对象生成默认构造函数、复制构造、赋值算符以及析构函数。

另外，C++也为所有类定义了诸如new，delete这样的运算符。当程序员有需要时，可以重载这部分函数，但可能无法精确控制默认函数的生成行为。

default函数作用于类的特殊成员函数，为其自动生成默认的函数定义体，提高代码的执行效率。

#include <iostream>
using namespace std;

class A {    // 未定义构造函数，编译器生成一个默认的构造函数
    public:
        int get() {
            return a;
        }
    private:
        int a;
};

int main() {
    A A1;  //可以实例化对象，类有默认构造函数
    cout << A1.get() << endl;
    return 0;
}
//输出：0

但当类中含有构造函数时，类不会为其生成默认构造函数：

#include <iostream>
using namespace std;

class A {
    public:
        A(int pa) {
            a = pa;    // 定义了一个特定的构造函数，则不生成默认构造函数
        }
        int get() {
            return a;
        }
    private:
        int a;
};

int main() {
    A A1;  //类没有默认构造函数  编译失败
    A A2(10);  // 使用特定的构造函数
    cout << A2.get() << endl;
    return 0;
}

可以让编译器自动生成特殊函数来代替手动编写的特殊成员函数。

class A {
    public:
        A() {}; // 如果想要使用默认构造函数时，必须显示定义。
        A(int pa) {
            a = pa;
        }
        int get() {
            return a;
        }
    private:
        int a;
};
//此时输出10

C++11标准引入了一个新特性：default函数。只需在类的特殊成员函数声明后加上=default;，就可将该函数声明为defaulted函数，编译器将为显式声明的default函数自动生成函数体。

class A {
    public:
        A() = default ; // 自动生成函数体。
        A(int pa) {
            a = pa;
        }
        int get() {
            return a;
        }
    private:
        int a;
};

default函数仅适用于类的特殊成员函数，且该特殊成员函数没有默认参数

default函数既可以在类内声明类外定义，也可以在类内直接定义。

class A {
    public:
        A() = default;   //自动生成函数体，类内定义
//A(double d) = default;  //错误：类型对于默认构造函数无效
        A(int a) {
            this->a = a;
        }
        A(const A &it);
//int get() = default;  //错误：=default只能出现在默认构造函数、析构函数、复制构造函数、赋值运算符中
    private:
        int a;
};

A::A(const A &it) = default;  //类内声明，类外定义

Delete

在C++11中，当我们定义一个类的成员函数时，如果后面使用=delete去修饰，那么就表示这个函数被定义为deleted，也就意味着这个成员函数不能再被调用，否则就会出错。

class TestClass {
    public:
        int func(int data) = delete;
};

int main(void) {
    TestClass obj;
    obj.func(100);  // 编译出错
    return 0;
}
//报错：[错误] use of deleted 函数 'int TestClass::func(int)'

在C++11之前，当我们希望一个类不能被拷贝，就会把构造函数定义为private，但是在C++11里就不需要这样做了，只需要在构造函数后面加上=delete来修饰下就可以了。

另一个应用：

#include <cstdio>

class TestClass {
    public:
        void func(int data) {
            printf("data: %d\n", data);
        }
};

int main(void) {
    TestClass obj;
    obj.func(100);
    obj.func(100.0);
    return 0;
}
//输出：data: 100
//data: 100

当我们把100.0传给obj.func()时，发生了隐式类型转换，由double转为了int，有时我们不希望发生这样的转换，我们就是希望传进来的参数和规定的类型一致，那么此时可以使用=delete来达到这个目的：

#include <cstdio>

class TestClass {
    public:
        void func(int data) {
            printf("data: %d\n", data);
        }
        void func(double data) = delete;
};

int main(void) {
    TestClass obj;
    obj.func(100);
    obj.func(100.0); // 编译出错
    return 0;
}
//报错：[错误] use of deleted 函数 'void TestClass::func(double)'

右值引用

c++中引入了右值引用和移动语义，可以避免无谓的复制，提高程序性能。

左值是可以放在赋值号左边可以被赋值的值，左值必须要在内存中有实体。

右值是在赋值号右边取出，值赋给其他变量的值，右值可以在内存也可以在CPU寄存器。

引用：变量的别名。

引用的基本规则：声明引用的时候必须初始化，且一旦绑定，不可把引用绑定到其他对象；即引用必须初始化，不能对引用重定义；对引用的一切操作，就相当于对原对象的操作。

C++03引用只能对左值，不能对右值。

int num = 10;
int &b = num; //正确
int &c = 10; //错误
//报错：[错误] cannot bind non-const lvalue reference of type 'int&' 到 an rvalue of type 'int'

但是常量引用，对左值右值都可以。

int num = 10;
const int &b = num;
const int &c = 10;

右值往往是没有名称的，因此要使用它只能借助引用的方式。

C++11标准新引入了另一种引用方式，称为右值引用，用&&表示。

和声明左值引用一样，右值引用也必须立即进行初始化操作，且只能使用右值进行初始化，比如：

int num = 10;
int &&a = num;   //右值引用不能初始化为左值
int &&a = 10;
a = 20; // 可以修改
const int &&b = 10;
b = 20; // 不可以修改

深拷贝与浅拷贝

浅拷贝

#include <iostream>
using namespace std;

class Test {
    public:
        int i;
        Test(int pi) {
            i = pi;
        }
};

int main() {
    Test a(10);
    Test b = a;  // 默认的拷贝构造
    cout << b.i << endl;     // 10
    a.i = 11;   // a 更新，
    cout << b.i << endl;     // 10    b未更新，符合直观
}

浅拷贝有时不能反映意图：

#include <iostream>
using namespace std;

class Test {
    public:
        int i;
        int *ptr;
        Test(int pi) {
            i = pi;
            ptr = new int(pi * 2);
        }
};

int main() {
    Test a(10);
    Test b = a;	  // 默认的拷贝构造
    cout << *b.ptr << endl;  // 20，看起来没问题，但实际上是拷贝了地址
    // 即：b.ptr==a.ptr. 这可能非本意
    *a.ptr = 22;   // a更新
    cout << *b.ptr << endl;    // 22. b也跟着跟新！
}

深拷贝

#include <iostream>
using namespace std;

class Test {
    public:
        int i;
        int *ptr;
        Test(int pi) {
            i = pi;
            ptr = new int(pi * 2);
        }
        Test(const Test &t) {
            i = t.i;
            ptr = new int(*t.ptr);
        }
};

int main() {
    Test a(10);
    Test b = a;	// 自定义的拷贝构造（深拷贝）
    cout << *b.ptr << endl;   // 20.但只是*b.ptr=*a.ptr而b.ptr！=a.ptr
    *a.ptr = 22;    // a更新了
    cout << *b.ptr << endl;    //  20.b是独立于a的
}

如果成员有指针则往往需要深拷贝。

移动拷贝

#include <iostream>
using namespace std;

class demo {
    public:
        demo() : num(new int(0)) {
            cout << "construct!" << endl;
        }
        //拷贝构造函数
        demo(const demo &d) : num(new int(*d.num)) {
            cout << "copy construct!" << endl;
        }
        ~demo() {
            cout << "class destruct!" << endl;
        }
    private:
        int *num;
};

demo get_demo() {
    return demo();
}

int main() {
    demo a = get_demo();
    return 0;
}

//完整的输出：
//construct!              <-- 执行 demo()
//copy construct!         <-- 执行 return demo()
//class destruct!         <-- 销毁 demo() 产生的匿名对象
//copy construct!         <-- 执行 a = get_demo()
//class destruct!         <-- 销毁 get_demo() 返回的临时对象
//class destruct!         <-- 销毁 a

//编译器优化后：
//construct!
//class destruct!

利用拷贝构造函数实现对a对象的初始化，底层实际上进行了2次拷贝（而且是深拷贝）操作。

对于仅申请少量堆空间的临时对象来说，深拷贝的执行效率依旧可以接受。

如果临时对象中的指针成员申请了大量的堆空间，2次深拷贝操作会影响a对象初始化的执行效率。

当类中包含指针类型的成员变量，使用其它对象来初始化同类对象时，怎样才能避免深拷贝导致的效率问题？

C++11标准引入了解决方案，该标准中引入了右值引用的语法，借助它可以实现移动语义。

移动构造函数

移动语义：以移动而非深拷贝的方式初始化含有指针成员的类对象，将其他对象（通常是临时对象）拥有的内存资源“移为已用”。

#include <iostream>
using namespace std;

class demo {
    public:
        demo(): num(new int(0)) {
            cout << "construct!" << endl;
        }
        demo(const demo &d): num(new int(*d.num)) {
            cout << "copy construct!" << endl;
        }
        //添加移动构造函数
        demo(demo &&d): num(d.num) {
            d.num = NULL;
            cout << "move construct!" << endl;
        }
        ~demo() {
            cout << "class destruct!" << endl;
        }
    private:
        int *num;
};

demo get_demo() {
    return demo();
}

int main() {
    demo a = get_demo();
    return 0;
}
//执行过程：
//construct!
//move construct!
//class destruct!
//move construct!
//class destruct!
//class destruct!

//输出：
//construct!
//class destruct!

函数使用右值引用形式的参数，又称为移动构造函数。

在此构造函数中，num指针变量采用的是浅拷贝的复制方式，同时在函数内部重置了d.num，有效避免了“同一块对空间被释放多次”情况的发生。

使用临时对象初始化a对象过程中产生的2次拷贝操作，都转由移动构造函数完成。

非const右值引用只能操作右值，程序执行结果中产生的临时对象（例如函数返回值、lambda表达式等）既无名称也无法获取其存储地址，所以属于右值。

当类中同时包含拷贝构造函数和移动构造函数时，如果使用临时对象初始化当前类的对象，编译器会优先调用移动构造函数来完成此操作。只有当类中没有合适的移动构造函数时，编译器才会退而求其次，调用拷贝构造函数。

在实际开发中，通常在类中自定义移动构造函数的同时，会再为其自定义一个适当的拷贝构造函数：当利用右值初始化类对象时，会调用移动构造函数；使用左值（非右值）初始化类对象时，会调用拷贝构造函数。

如果使用左值初始化同类对象，但也想调用移动构造函数完成，如何实现？

默认情况下，左值初始化同类对象只能通过拷贝构造函数完成，如果想调用移动构造函数，则必须使用右值进行初始化。C++11标准中为了满足用户使用左值初始化同类对象时也通过移动构造函数完成的需求，新引入了 std::move()函数，它可以将左值强制转换成对应的右值，由此便可以使用移动构造函数。

move(arg)

其中，arg表示指定的左值对象。该函数会返回arg对象的右值形式。

#include <iostream>
using namespace std;

class movedemo {
    public:
        movedemo() : num(new int(0)) {
            cout << "construct!" << endl;
        }
        //拷贝构造函数
        movedemo(const movedemo &d) : num(new int(*d.num)) {
            cout << "copy construct!" << endl;
        }
        //移动构造函数
        movedemo(movedemo &&d) : num(d.num) {
            d.num = NULL;
            cout << "move construct!" << endl;
        }
        int *num;
};

int main() {
    movedemo demo;
    cout << "demo2:\n";
    movedemo demo2 = demo;
    cout << *demo2.num << endl;   //可以执行
    cout << "demo3:\n";
    movedemo demo3 = std::move(demo);
    //此时 demo.num = NULL，因此下面代码会报运行时错误
    //cout << *demo.num << endl;
    return 0;
}
//输出：
//construct!
//demo2:
//copy construct!
//0
//demo3:
//move construct!

通过观察程序的输出结果，以及对比demo2和demo3初始化操作不难得知，demo对象作为左值，直接用于初始化 demo2对象，其底层调用的是拷贝构造函数；而通过调用move()函数可以得到demo对象的右值形式，用其初始化demo3对象，编译器会优先调用移动构造函数。

注意，调用拷贝构造函数，并不影响demo对象，但如果调用移动构造函数，由于函数内部会重置demo.num指针的指向为NULL，所以程序中后面代码会导致程序运行时发生错误。

完美转发（Perfect Forward）

完美转发指的是函数模板可以将自己的参数“完美”地转发给内部调用的其它函数。所谓完美，即不仅能准确地转发参数的值，还能保证被转发参数的左、右值属性不变。
