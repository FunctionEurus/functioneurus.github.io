#8 异常处理

2022-06-03
  • C++

异常机制

异常指运行时出现的不正常现象，如内存耗尽、意外的非法输入、硬件故障……

当程序代码检查到异常时，需要错误检测程序的干预，称之为异常处理机制：把控制权转到可处理该问题的程序，指出具体出现了什么问题，并提供一些附加信息。

使用异常处理，程序各部分能够就程序执行期间出现的问题相互通信，并处理这些问题。

异常处理语句

通常使用try块来处理异常。

try语句块以try关键字开始，并以一个或多个catch子句结束。在try块中执行的代码所抛出（throw）的异常， 通常会被其中一个catch子句处理。也称为处理代码。

try块的通用语法形式是：

try  {
    program - statements
} catch (exception - specifier) {
    handler - statements
} catch (exception - specifier) {
    handler - statements
}
//...

Throw

异常对象可以主动引发。

语法：throw表达式，表达式包含一个对象或指针。

exception头文件定义了异常类，如exception类。异常类用来在throw和相应的处理程序之间（catch）之间传递信息。异常类以类似于将实参传递给函数的方式抛出和捕获。

程序执行到throw的时候，跟在throw后面的语句不再执行， 控制权将从throw转移到匹配的catch语句，该catch可以是同一函数中局部的catch，也可以是在直接或间接调用该发生异常的函数的另一个函数。

void f2() {
    cout << "enter f2" << endl;
    throw new exception();
    cout << "exit f2" << endl;
}

void f1() {
    cout << "enter f1" << endl;
    f2();
    cout << "exit f1" << endl;
}

int main() {
    cout << "enter main" << endl;
    f1();
    cout << "exit main" << endl;
    return 0;
}

//运行结果：
//enter main
//enter f1
//enter f2 
//程序出现异常！
//在整个程序中都未找到处理程序。
//程序异常终止。

栈展开（Stack unwinding）

抛出异常的时候，将暂停当前函数的执行，开始查找匹配的catch子句。

首先检查throw本身是否在try块内部，如果是，检查与该catch相关的catch子句，看是否其中之一与抛出对象相匹配。如果找到匹配的catch，就处理异常；如果找不到，就退出当前函数并继续在调用函数中查找（Call stack）未捕获的异常终止程序。沿嵌套函数调用链继续向上，直到为异常找到一个catch子句。

只要找到能够处理所遇异常的catch子句， 就进入该catch子句，并在该处理代码中继续执行。此过程称为栈展开stack unwinding。如上栈展开的过程为f2->f1->main。

当catch结束的时候，在紧接在与该try块相关的最后一个catch子句之后的点继续执行。

内存泄漏

栈展开期间，会提早退出包含throw的函数和调用链中可能的其他函数，这些函数已经可能已经创建了局部对象。因异常而退出函数时，编译器会保证适当地撤销局部对象，但是动态分配的内存不会被释放，因而可能会造成内存泄漏。

void f2() {
    cout << "enter f2" << endl;
    int i = 3;   // 自动变量没关系
    int *pi = new int(3);   // 动态分配
    throw new exception();
    cout << "exit f2" << endl;
    delete pi;   //不会执行，内存泄漏
}

void f1() {
    int j = 4;
    int *pj = new int(4); // 动态分配
    cout << "enter f1" << endl;
    f2();
    cout << "exit f1" << endl;
    delete pj;   //不会执行，内存泄漏
}

int main() {
    cout << "enter main" << endl;
    f1();
    cout << "exit main" << endl;
    return 0;
}

捕获异常

catch子句中的异常说明符决定了处理代码能够捕获的异常种类。

在查找匹配的catch期间，找到的catch不一定是与异常最匹配的那个catch，系统将选中第一个找到的可以处理该异常的catch。因此， 在catch子句列表中，最特殊的catch应最先出现。若带有因继承而相关的类型的多个catch子句，则必须从最低派生类类到最高派生类型排序。

进入catch的时候，用异常对象初始化catch的形参。

//将f2 改写为：

void f2() {
    cout << "enter f2" << endl;
    try {
        cout << "enter try " << endl;
        throw new exception();
        cout << "exit try" << endl; // 这一句不会被执行
    } catch (exception *pe) {
        cout << "exception caught and handled" << endl;
    }
    cout << "exit f2" << endl;
}

//执行情况为：
//enter main
//enter f1
//enter f2 
//enter try
//exception caught and handled
//exit f2
//exit f1
//exit main

将程序稍作调整，可以更清楚看清捕获异常的过程：

void f2() {
    cout << "enter f2" << endl;
    throw new exception();
    cout << "exit f2" << endl;
}

void f1() {
    cout << "enter f1" << endl;
    try {
        cout << "enter try " << endl;
        f2();  // 调用一个有可能抛出异常的函数
        cout << "exit try" << endl;
    } catch (exception *pe) {
        cout << "exception caught and handled" << endl;
    }
    cout << "exit f1" << endl;
}

//异常在f2引发，而在f1处理。
//程序在catch语句后恢复正常运行。

异常信息的传递

异常类用来在throw和相应的catch之间传递，异常以类似于将实参传递给函数的方式抛出和捕获。

try {
    int i = 1; // 栈中的变量
    throw i;
} catch (int  e) { // 拷贝到此处int &e也行，int* e则出错
    e++;
    cout << e << endl;
}

try {
    int *pi = new int(1);
    throw pi;
} catch (int *e) {
    (*e)++;
    cout << *e << endl;
    delete e;
}

重新抛出

有可能单个catch不能完全处理一个异常。

在进行了一些校正行动之后，catch可能确定该异常必须由函数调用链中更上层的函数来处理， 此时可以重新抛出异常，将其传递给函数调用链中更上层的函数。

语法：throw;

使用场合：只能出现在catch语句内部， 或者从catch调用的函数中。

捕获所有异常

一般总是为特定的异常提供特定catch子句。

有时可能不知道会抛出什么异常，使用以下语句可以捕获所有异常 ：

// matches any exception that might be thrown

catch (...) {
    // place our code here
}

例子

#include <iostream>
using namespace std;

int Div(int x, int y) {
    if (y == 0)
        throw y;
    return x / y;
}

int  main() {
    try {
        cout << "5/2=" << Div(5, 2) << endl;
        cout << "8/0=" << Div(8, 0) << endl;
        cout << "7/1=" << Div(7, 1) << endl;
    } catch (int) { //如改为exception，则无法捕捉。
        cout << "except of deviding zero.\n";
    } catch (...) {
        cout << "any exception" << endl;
    }
    cout << "after try catch" << endl;
}

//输出为：
//5/2=2
//8/0=except of deviding zero.
//after try catch

class MyEx : public exception {
    public:
        int i;
        MyEx(int pi): exception("abc") {
            i = pi;
        }
};

int main() {
    try {
        throw  MyEx(10);
    } catch (MyEx myex) {
        cout << myex.i << myex.what() << endl;
    }
}
//运行的时候报错
//[错误] no matching 函数 对于 call 到 'std::exception::exception(const char [4])'

Noexcept

void f() throw() { // 表明函数内部不会发生异常
    throw 1; // 如果发生了，编译给出警告
}

//指定可能发生的异常。
void excpt_func() throw(int, double)
{ 
    ...
}

void f() noexcept {
    throw 1;
}
//实际上后面两段我没看懂（哭

noexcept有两种形式，一是简单地在函数声明后加上noexcept关键字，如void excpt_func() noexcept; 。二是接受一个常量表达式作为参数，如void excpt_func() noexcept (常量表达式); 。常量表达式的结果会被转换成一个bool类型的值。该值为true，表示函数不会抛出异常，反之，则有可能抛出异常。这里，不带常量表达式的noexcept相当于声明了noexcept(true)，即不会抛出异常。


#include <iostream>
using namespace std;
void Throw() {
    throw 1;
}

void NoBlockThrow() {
    Throw();
}

void BlockThrow() noexcept {
    Throw();
}

int main() {
    try {
        Throw();
    } catch (...) {
        cout << "Found throw." << endl;     // Found throw.
    }
    try {
        NoBlockThrow();
    } catch (...) {
        cout << "Throw is not blocked." << endl;    // Throw is not blocked.
    }
    try {
        BlockThrow();   // terminate called after throwing an instance of 'int'
    } catch (...) {
        cout << "Found throw 1." << endl;
    }
}

//输出为：
//Found throw.
//Throw is not blocked.
//terminate called after throwing an instance of 'int'

C++11使用noexcept

noexcept表示其修饰的函数不会抛出异常。与throw()不同的是，在C++11中如果noexcept修饰的函数抛出了异常，编译器可以选择直接调用std::terminate()函数来终止程序的运行，这比基于异常机制的throw()在效率上会更高。因为异常机制会带来一些额外开销，比如函数抛出异常，会导致函数栈被依次地展开（unwind），并调用已构造的自动变量的析构函数等。
