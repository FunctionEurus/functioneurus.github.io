#5 运算符重载

2022-05-30
  • C++

C语言中操作符重载的例子：

int a = 3, b = 5;
int c = a * b; //表示乘积
int *p = &a; // *表示p是一个指针
int i = *p;  //*作用在地址上表示该地址所记录的值

一个运算符可以有多种含义，编译器会根据操作符的操作数自动判断对操作符赋予哪一种含义。

C++编译器把运算符看成一个函数：

c = a * b //解释为  c = operator*(a,b);
i = *p   //解释为 i = operator*(p);

operator*()，称为运算符函数，这个函数被重载。

C++允许对操作符函数进行重载，简称为操作符重载。

class Complex  {
    public:
        double real;//实部
        double imag;//虚部
        Complex(double r = 0, double i = 0) {
            real = r;
            imag = i;
        }
        void show() {
            cout << "real= " << real << " " << "imag= " << imag << endl;
        }
};

//以下重载运算符函数operator+,此函数的参数为两个Complex,且在计算过程中不会被改变,返回值也是一个Complex:

Complex operator+(const Complex &op1, const Complex &op2) {
    Complex t;
    t.real = op1.real + op2.real ; //读取,设置了Complex的公有的数据成员
    t.imag = op1.imag + op2.imag ;
    return t;

}  // 注意：这里使用了默认的拷贝构造函数。
int main() {
    Complex ob1(2, 3);
    Complex ob2(0.5, 0.6);
    Complex sum;
    sum = ob1 + ob2; //使用运算符函数sum=operator+(ob1,ob2);
    sum.show ();
}

以上代码对+进行了重载，使之可以进行Complex类的运算，但它与Complex并没有直接的联系，故只能访问Complex的公共成员。Complex的两个数据成员都是公共的，但是这不符合OOP的封装原则，存在安全隐患。

一般说来， Complex的两个数据成员都应该是私有的，但此时+就不能访问它了。为此，有两种解决方法：

把operaor+()运算符函数作为类的成员函数，以使得其可以访问私有的成员，称为成员操作符函数；把operaor+()运算符函数作为类的友元函数，称为友元操作符函数。

class Complex {
    private:  //注意这里的数据成员是私有的
        double real;//实部
        double imag;//虚部
    public:
        Complex(double r = 0, double i = 0) {
            real = r;
            imag = i;
        }
        void show() {
            cout << "real= " << real << " " << "imag= " << imag << endl;
        }
        Complex operator+(const Complex &op); //成员操作符函数声明
};

//重载了成员操作符函数,注意参数只有一个!
Complex Complex::operator +(const Complex &op) { // 定义
    Complex t;
    t.real = real + op.real ; // 即t.real= this->real+op.real
    t.imag = imag + op.imag ;
    return t;
}

int main() {
    Complex ob1(2, 3);
    Complex ob2(0.5, 0.6);
    Complex sum;
    sum = ob1 + ob2; //使用成员运算符函数: sum=ob1.operator+(ob2)
    sum.show ();
}

使用友元操作符函数重载

在类定义中加上：

friend Complex operator +(int i, Complex &op );

具体实现：

Complex operator +(int i, Complex &op ) { // 也没有this指针
    Complex t;
    t.real = op.real + i ;
    t.imag = op.imag + i;
    // 为了展示用法,这里的加法为整数与复数相加，且实现部分为对实部和虚部同时加上一个整数. 其他正常意义上的加法需要重新实现。
    return t;
};
//调用时：
Complex ob1(2, 3);
sum = 3 + ob1;
//sum将为(5,6).

++及–运算符

//前缀形式：
Complex &operator++() {
    this->real++;
    this->imag++;
    return *this;
}

//调用：
Complex c(2, 3);
++c;

//后缀形式,稍显生硬
Complex &operator++(int ) { // 此处并不需要给出参数名 且此额外参数必须为int 类型
    this->real++;
    this->imag++;
    return *this;
}
//调用：
Complex c(2, 3);
c++;

<<及>>运算符

在C++中，左移运算符<<可以和cout一起用于输出，因此也常被称为“流插入运算符”或者“输出运算符”。实际上，<<本来没有这样的功能，之所以能和cout一起使用，是因为被重载了。

cout是ostream类的对象。ostream类和cout都是在头文件<iostream>中声明的。

ostream类将<<重载为成员函数，而且重载了多次。为了使cout<<"Star War"能够成立，ostream类需要将<<进行如下重载：

ostream &ostream::operator << (const char *s) {
    //输出s的功能代码
    return * this;
}

为了使cout<<5;能够成立，ostream类还需要将<<进行如下重载：

ostream &ostream::operator << (int n) {
    //输出n的代码
    return *this;
}

重载函数的返回值类型为ostream的引用，并且函数返回*this，就使得cout<<"Star War"<<5;能够成立。

有了上面的重载，cout<<"Star War"<<5;就等价于：

cout.operator << ("Star War") ).operator << (5);

对Complex的输出

我们希望对Complex的输出和其他的如字符串、整数等统一的方式进行。

但不能通过修改ostream类和istream类以达到该目的（标准库的类）。

也不能将<<和>>重载为Complex类的成员运算符函数，应与cout的其他用法保持一致。<<是ostream类的成员。

故只能将<<和>>重载为全局运算符函数。由于这两个函数需要访问Complex类的私有成员，因此在Complex类定义中将它们声明为友元。

friend ostream &operator<< (ostream &os, const Complex &c) {
    os << "real is " << c.real << " imag is " << c.imag << endl;
    return os;
}
//使用方法：
Complex c(2, 3);
cout << c << endl;

//输入也是一样：
friend istream &operator>>(istream &is, Complex &c);

对[]运算符的重载

使用[]可以方便访问数组的元素。对于某些集合类，对[]重载可以方便访问其内部成员。

#include <iostream>
#include <string>
using namespace std;

class FootballTeam {
        string players[11];  // 私有的集合类成员
    public:
        FootballTeam() {
            for (int i = 0; i < 11; i++)
                players[i] = "player" + to_string(i);
        }
        string operator[ ] (int idx) { // 重载下标运算符
            return players[idx];
        }
};

int main() {
    FootballTeam ft;
    cout << ft[3] << endl;	 // 使用更直观
    return 0;
}
//输出player3\n

对!的重载

class Foo {
        int number{ 0 };
    public:
        Foo(int c) {
            number = c;
        }
        Foo() = default;
        Foo &operator !() {  // 重载取反运算符
            number = -number;
            return (*this);
        }
        std::string toString() {
            return ("the number is " + std::to_string(number) + "\n");
        }
};

调用运算符()

可以为类类型的对象重载函数调用操作符()，一般为表示操作的类重载调用操作符。例如，可以定义名为absInt的结构，该结构封装将int类型的值转换为绝对值的操作：

struct absInt {
    int operator() (int val) {
        return val < 0 ? -val : val;
    }

    int i = -42;
    absInt absObj; // object that defines function call operator
    unsigned int ui = absObj(i); // calls absInt::operator(int)

调用运算符函数的使用，看起来像是函数调用。

absObj是一个对象而不是函数，但是仍然可以“调用”之。

函数调用操作符必须声明为成员函数，且可以重载之。

函数对象

定义了调用操作符的类，其对象常称为函数对象（function object），即它们是行为类似函数的对象，又称为仿函数。
